<Update label="2026-03-02" description="v1.2.0" tags={["Feature", "Breaking", "Channel"]}>
  ## NanoClaw v1.2.0

  Major release introducing pluggable multi-channel architecture. WhatsApp is no longer hardcoded — all channels (WhatsApp, Telegram, Discord, Slack, Gmail) now self-register via a channel registry.

  ### Features
  - **Channel registry**: Channels self-register at module load time via `registerChannel()` factory pattern
  - **`isMain` flag**: Explicit boolean replaces folder-name-based main group detection
  - **Channel-prefixed group folders**: Groups now use `whatsapp_main`, `telegram_family-chat` convention to prevent cross-channel collisions
  - **Unconfigured channel warnings**: Channels with missing credentials emit WARN logs naming the exact missing variable instead of being silently skipped

  ### Breaking changes
  - **WhatsApp moved to skill**: WhatsApp is no longer part of core — apply it with `/add-whatsapp`
  - **`ENABLED_CHANNELS` removed**: The orchestrator now uses `getRegisteredChannelNames()` from the channel registry; channels are detected by credential presence
  - **All channel skills simplified**: Discord, Slack, Gmail, Telegram skills now use the same self-registration pattern — no more `*_ONLY` flags

  ### Migration
  Existing WhatsApp users: run `/add-whatsapp` in Claude Code CLI after updating to re-apply the WhatsApp channel as a skill.
</Update>
